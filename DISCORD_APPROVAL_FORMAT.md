# DISCORD_APPROVAL_FORMAT.md — Discord 승인 카드 기준

## 목적
톡톡 / 주문문의 / 상품문의 초안을 Discord 안에서 검토하고 승인하기 위한 보고 포맷을 정의한다.

## 기본 원칙
- 승인 UI는 별도 웹보다 Discord를 우선한다.
- 승인 요청에는 초안만 보내지 말고, 최근 대화 맥락과 핵심 주문/배송 정보를 함께 보낸다.
- 대표 승인 전에는 실제 반영하지 않는다.
- 감사/확인/알겠습니다 같은 종결성 답신은 승인 요청 대상으로 올리지 않는다.
- 최종 목표는 텍스트 명령보다 Discord 버튼 UI(`승인`, `보류`, `수정요청`)를 우선 사용하고, 텍스트 명령은 백업 수단으로 둔다.
- 승인 카드 생성과 버튼 payload 생성은 웹서버가 아니라 로컬 자동화에서 수행한다.

## 승인 요청 메시지 필수 항목
1. 문의 번호
2. 채널 (톡톡 / 주문문의 / 상품문의)
3. 문의 유형 (배송문의 / 취소문의 / 옵션문의 / 통관문의 등)
4. 고객명
5. 상품명
6. 주문번호 / 상품주문번호
7. 송장번호 (있는 경우)
8. 배송 상태 / 통관예상 / 배송예상 (있는 경우)
9. 최근 대화 3~5개
10. 생성 초안
11. 승인 입력 예시

## 권장 메시지 구조
```text
[1번] 톡톡 / 배송문의 / 장형석
상품: 테슬라 모델Y 주니퍼 정품 차박매트 에어 매트 메트리스
주문번호: 2026033026786761
상품주문번호: 2026033073960541
송장번호: 303593965841
현재상태: 통관대기
통관예상: 04월 06일 (월)
배송예상: 04월 07일 (화)

[최근 대화]
- 고객 18:55 스티커를 전송했습니다.
- 고객 18:56 언제쯤 받을 수 있을까요?
- 상담사 22:20 배송 현황 안내드립니다...

[초안]
안녕하세요, 고객님 😊
...

승인 예시:
- 1번 승인
- 1번 보류
- 1번 수정: 마지막 문장만 더 짧게
```

## 대표 승인 명령 규칙
- `1번 승인`
- `1,2번 승인`
- `1번 보류`
- `1번 수정: 마지막 문장만 더 짧게`
- `전체 보류`

## Discord 버튼 UI 목표 형태
- 버튼 1: `승인`
- 버튼 2: `보류`
- 버튼 3: `수정요청`
- 버튼 클릭 시 승인 객체의 상태를 각각 `approved`, `held`, `revision_requested`로 변경한다.
- 텍스트 명령은 버튼 장애 시 백업 경로로 유지한다.
- 버튼 클릭 이벤트만으로 approval 대상을 특정하기 어렵다면, 카드 본문과 버튼 라벨에 `승인코드`(예: `dd6075`)를 함께 노출하고 로컬 approval 파일의 `shortCode`와 매칭해 처리한다.

## 버튼 클릭 후 처리 규칙
- 버튼 클릭이 들어오면 `승인코드`와 액션(`승인`/`보류`/`수정요청`)을 추출한다.
- 즉시 아래 명령으로 로컬 approval 상태를 반영한다.

```bash
node ./scripts/local-approval-action.mjs <승인코드> <approve|hold|revise> 대표님
```

- 위 명령 결과가 `approved` 이면, 반환된 `customerName`, `productName`, `draft`를 사용해 같은 흐름 안에서 바로 톡톡 파트너센터에 반영한다.
- `held`/`revision_requested` 이면 실제 반영은 하지 않는다.

## 승인 큐 운영 원칙
- `approved` 는 곧바로 `로컬 처리 큐 적재`를 의미한다.
- 대표는 완료 보고를 기다리지 않고 다음 승인 카드 버튼을 계속 누를 수 있다.
- 로컬 브라우저 자동화는 큐에 쌓인 승인 건을 한 건씩 순차 처리한다.
- 각 건 완료 시마다 부분 캡처 첨부 완료 보고를 따로 올린다.

## 후속 처리
- `승인` → 로컬 브라우저 자동화가 실제 톡톡/마켓 화면에 반영
- `보류` → 후속 확인 전까지 대기
- `수정` → 초안 재생성 후 다시 승인 요청

## 톡톡 배송문의 추가 규칙
- 퀵스타 로그인 상태를 먼저 확인한다.
- 주문정보 팝업은 정보 파싱 후 바로 닫는다.
- Quickstar Tracker 조건 검색 결과가 모두 표시될 때까지 기다린 뒤 초안을 생성한다.
- 최근 대화 맥락을 보고 감사/종결성 답신이면 승인 요청을 올리지 않는다.

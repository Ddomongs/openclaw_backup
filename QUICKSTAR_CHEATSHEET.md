# QUICKSTAR_CHEATSHEET.md — 배송조회 치트시트

## 목적
Quickstar Tracker 크롬 플러그인의 핵심 동작만 빠르게 확인하기 위한 운영 치트시트.
원문 기술문서 대신 실무에서 바로 찾는 정보만 남긴다.

---

## 1. 입력값 자동 감지 규칙
| 입력 형식 | 인식 대상 |
|---|---|
| 숫자 12자리 | 국내 운송장 |
| `GR` 시작 | 그룹번호 |
| `010-` 시작 | 수취인 연락처 |
| `YT`, `SF`, `JD`, `77`, `78` 시작 | 해외트래킹 |
| 숫자 16자리 이상 | 해외주문번호 |
| 한글 2~4자 | 수취인 |
| 영문 시작 문자열 | 상품명 |

---

## 2. 조회 흐름
1. 입력값 자동 감지
2. Quickstar / 7customs / CJ대한통운 병렬 조회
3. 통관 상태 / 반입 예정 / 국내 배송 상태 조합
4. 고객 안내 초안 생성

### 조회 소스 역할
- **Quickstar**: 해외/입고/반입 관련 상태
- **7customs**: 통관 상태 및 통관 예상
- **CJ대한통운**: 국내 배송 상태 / 배달기사 정보

---

## 3. 고객 안내 원칙
- 해외창고 / 통관 / 국내배송 단계를 섞지 않는다.
- 내부 물류 상태를 그대로 노출하지 않고 고객 기준으로 정리한다.
- 초기 내부 상태(결제대기, 출고준비 등)는 고객에게 과도하게 세분화하지 않는다.
- tracking.tipoasis 링크를 우선 사용한다.

---

## 4. 답변 초안에 자주 들어가는 항목
- 현재 상태
- 통관 예상일
- 국내 반입 예정
- 국내 배송 상태
- 배송 예상일
- 배송조회 링크

---

## 5. 수정 포인트 찾는 법
- `content.js`
  - 검색 카테고리 정의
  - 입력값 자동 감지 정규식
  - 결과 렌더링 / 답변 초안 생성
- `background.js`
  - Quickstar / 7customs / CJ 조회 로직
  - DOM 스크래핑 / fetch 프록시
- `options.js`
  - 검색 범위, 계정, 카테고리 순서 설정

---

## 6. 대표님 실무에서 자주 쓰는 경우
- 운송장 번호로 배송 상태 확인
- 통관 지연 여부 확인
- 고객 문의 답변 초안 생성
- 수취인 이름 / 연락처 / 해외트래킹으로 역추적

---

## 7. 기억할 것
- 배송조회 목적은 단순 상태 확인이 아니라 **CS 응답 속도 향상**이다.
- 조회 결과는 고객이 이해할 수 있게 재정리해서 안내한다.
- 통관/배송 단계 오안내가 가장 큰 실수 포인트다.

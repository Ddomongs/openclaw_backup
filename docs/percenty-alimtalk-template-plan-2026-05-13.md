- 기준 JS: `message---f32c368a-0dc9-465f-8bd5-dab7ce6b49b5.txt`

- 현재 구조: 퍼센티/퀵스타 주문 데이터 추출 → 문자 미리보기 생성 → GM 저장소 큐 저장 → Google Messages 웹에서 반자동 SMS 발송

## 구현 구조 요약
1. `TEMPLATE_LABELS`, `SMS_TEMPLATES`는 알림톡 템플릿 정의의 원천으로 사용한다.
2. 알리고 신청용 문구는 `{변수}`를 `#{변수}`로 변환한다.
3. 유저스크립트에는 Aligo API 키를 넣지 않는다. 로컬/서버 중계 API에 `ALIGO_API_KEY`, `ALIGO_USER_ID`, `ALIGO_SENDERKEY`, `ALIGO_SENDER`를 환경변수로 둔다.
4. JS는 기존 `saveCurrent()` 큐 구조를 유지하되, 발송 단계만 `messages.google.com` 대신 중계 API 호출로 바꾼다.
5. `tpl_code` 매핑은 승인 후 `ALIMTALK_TEMPLATE_MAP[type] = tpl_code` 형태로 연결한다.

## 승인 리스크 / 수정 권장
- `{취소안내}`는 마켓별 긴 문구와 URL이 들어가는 동적 블록이라 알림톡 심사 리스크가 높다. 가능하면 템플릿을 마켓별로 분리하거나, 고정 문구로 단순화한다.
- 알림톡에는 광고/구매유도 표현이 없어야 한다. 현재 문구는 대부분 주문/통관/재고 안내성이라 가능성이 있으나, “가격인상”/“품절취소”는 취소 안내 목적임을 명확히 유지한다.
- 개인정보 최소화를 위해 `#{통관부호}` 전체값은 템플릿에 넣지 않는 것을 권장한다. 현재 10개 템플릿 본문에는 통관부호 전체값 변수는 직접 쓰이지 않는다.

## 카카오 알림톡 유형 가이드 검토 반영 (2026-05-13)

### 확인한 가이드 핵심
- 메시지 유형
  - 기본형: 고객에게 반드시 전달되어야 하는 정보성 안내에 적합.
  - 부가정보형: 본문 하단에 고정 부가정보를 붙이는 유형. 본문 1,300자 + 부가정보 500자 구조로 보임.
  - 채널추가형: 비광고성 메시지 하단에 채널 추가 문구를 붙이는 유형. 문구 수정 불가.
  - 복합형: 부가정보형 + 채널추가형 동시 사용.
- 강조 유형
  - 이미지형: 상단 고정 이미지 1개를 쓰는 유형. 정보성 목적/본문 관련 이미지에 한정, 광고성 이미지 불가, 강조표기형과 동시 사용 불가.
  - 강조표기형: 말풍선 상단에 핵심정보/보조문구를 크게 보여주는 유형. 고객이 반드시 인지해야 하는 상태 안내에 적합.
  - 아이템리스트형: 주문/결제/예약/배송 등 구조화된 항목을 리스트로 보여주는 유형. 단, 알리고 API 스펙에서 실제 지원 여부 확인 필요.
- 미리보기 메시지
  - 최대 40자, 변수 사용 불가, 채팅방 리스트/푸시 노출용.

### 이번 퍼센티/퀵스타 템플릿에 맞는 유형 판단

#### 1순위: 기본형(BA) + 강조유형 없음(NONE)
- 적용 대상: 10개 전체 기본 신청안.
- 이유: 현재 문구가 모두 주문/통관/재고/취소 관련 정보성 안내라 기본형이 가장 무난하고 승인 리스크가 낮다.
- 특히 `PCCC_OK`는 단순 확인 완료 안내라 기본형이 가장 적합하다.

#### 2순위: 강조표기형(TEXT)
- 적용 추천 대상:
  - `PCCC_NAME_MISMATCH`: 타이틀 `통관정보 확인 필요`, 보조문구 `성명 확인 필요`
  - `PCCC_PHONE_MISMATCH`: 타이틀 `통관정보 확인 필요`, 보조문구 `전화번호 확인 필요`
  - `PCCC_ZIP_MISMATCH`: 타이틀 `통관정보 확인 필요`, 보조문구 `우편번호 확인 필요`
  - `PCCC_REQUEST`: 타이틀 `통관부호 회신 필요`, 보조문구 `개인통관고유부호 미입력`
  - `STOCK_ALT_OPTION` / `STOCK_MULTI_ALT` / `PARTIAL_STOCK_OUT`: 타이틀 `옵션 확인 필요`, 보조문구 `재고 소진 안내`
  - `STOCK_CANCEL`: 타이틀 `주문 취소 안내`, 보조문구 `현지 재고 소진`
  - `PRICE_INCREASE`: 타이틀 `가격 변동 안내`, 보조문구 `출고 가능 여부 확인 필요`
- 이유: 고객이 조치해야 하는 유형에서 상단 강조가 효과적이다.
- 주의: 강조 타이틀/보조문구에는 개인정보, 통관부호 전체값, 과도한 광고성 문구를 넣지 않는다.

#### 3순위: 아이템리스트형
- 적용하면 좋은 대상:
  - 통관불일치 3종: 항목 `확인항목 / 주문마켓 / 상품명 / 배송주소 또는 연락처`
  - 재고부족/부분품절: 항목 `품절옵션 / 대체옵션 / 요청사항`
- 이유: 변수 항목이 많은 템플릿을 고객이 빠르게 이해하기 좋다.
- 단, 알리고 API 문서에서 확인한 `tpl_emtype` 예시는 `NONE`, `TEXT`, `IMAGE` 중심이라 아이템리스트형 지원 여부를 알리고 관리자 화면/API에서 재확인해야 한다. 지원이 불확실하면 1차 신청에는 제외한다.

#### 비추천: 이미지형
- 이번 10개 템플릿에는 비추천.
- 이유: 이미지가 템플릿마다 고정되어야 하고, 본문과 직접 관련된 정보성 이미지만 가능하다. 통관/재고 안내에 실익보다 심사 부담이 크다.

#### 비추천: 채널추가형/복합형
- 이번 10개 템플릿에는 비추천.
- 이유: 고객 CS 안내 목적과 채널 마케팅 수신 동의 목적이 섞일 수 있어 승인/민원 리스크가 생긴다.

### 최종 추천 신청 순서
1. 1차 신청: 10개 모두 `기본형 + 강조유형 없음`으로 신청.
2. 2차 개선: 반응이 중요한 8개만 `강조표기형`으로 별도 추가 신청.
   - 제외 후보: `PCCC_OK`는 기본형 유지.
3. 아이템리스트형은 알리고 관리자 화면에서 지원 여부 확인 후, 재고/통관 주요 템플릿만 별도 실험.

### 미리보기 메시지 제안

- `PCCC_PHONE_MISMATCH`: 통관정보 전화번호 확인 필요
- `PCCC_ZIP_MISMATCH`: 통관정보 우편번호 확인 필요

- `STOCK_CANCEL`: 품절로 인한 취소 안내

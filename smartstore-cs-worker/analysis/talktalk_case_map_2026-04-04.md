# 톡톡 케이스 맵 (2026-04-04)

기준 데이터: 최근 톡톡 대화 30건 패턴 분석
핵심: 상품 Q&A보다 배송/통관/배송일정 문의 비중이 높아 자동화 적합도가 더 높음

## 1. 우선순위 높은 케이스

### 1) shipping_eta_basic
- 질문 유형: 언제쯤 배송되나요 / 언제 도착하나요 / 며칠 걸리나요
- 톤: 짧은 즉답형
- 원칙: 배송일 확답보다 변동 가능성 안내 + 확인 후 재안내

### 2) shipping_eta_long
- 질문 유형: 일반 배송기간 문의 / 평균배송일 문의
- 톤: 설명형
- 원칙: 해외발송 + 통관 변수 설명

### 3) shipping_delay_apology
- 질문 유형: 오래 기다렸는데 / 아직도 발송 준비중 / 왜 안 오나요
- 톤: 사과 + 설명형
- 원칙: 사과 먼저, 진행상황 확인 후 재안내

### 4) tax_fee_clear
- 질문 유형: 관세 붙나요 / 추가금 있나요 / 포함금액인가요
- 톤: 짧은 즉답형
- 원칙: 내부 기준이 명확할 때만 단정

### 5) cancel_refund_route
- 질문 유형: 취소 / 환불 / 반품 / 교환
- 톤: 짧은 안내형
- 원칙: 주문 상태 확인 후 절차 안내

### 6) stock_check_needed
- 질문 유형: 재고 있나요 / 구할 수 있나요 / 입고되나요
- 톤: 설명형
- 원칙: 재고는 시점 변동이 커서 확인 후 안내

### 7) option_guidance_check
- 질문 유형: 옵션 / 색상 / 16:9 / 4:3 / 선택방법
- 톤: 설명형
- 원칙: 판매페이지 기준 + 확인 후 안내

## 2. 보조 케이스

### usage_check_needed
- 질문 유형: 사용법 / 설치 / 조립 / 어떻게
- 톤: 짧은 확인형

### default_check
- 질문 유형: 명확한 분류 불가
- 톤: 짧은 보류형

## 3. 운영 원칙

1. 톡톡은 상품 Q&A보다 배송문의 비중이 높으므로 자동화 우선 채널로 본다.
2. 확실한 사실(추가금 없음, 취소 경로 등)은 짧게 답한다.
3. 일정/통관/배송지연은 보수적으로 안내한다.
4. 오래 기다린 고객은 사과 문구를 먼저 둔다.
5. 재고/옵션은 확인 후 안내 패턴을 유지한다.
6. 위험 CS(환불금액, 보상, 분쟁)는 자동확답하지 않는다.

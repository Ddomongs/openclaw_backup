스토어별 CS 통합 점검 시간입니다.

대상 스토어: 또몽이네 스토어
상태 파일: `./tmp/cs-monitor-state.json`

반드시 아래 순서대로 진행하세요.

[1] 브라우저 준비
1. 먼저 `./scripts/browser-mcp/chrome-dev-approve-session.sh start`를 실행해 브라우저 attach 허용 세션을 시작합니다.
2. 이어서 `./scripts/browser-mcp/chrome-dev-approve-session.sh status`로 세션이 `running`인지 확인합니다. `running`이 아니면 한 번 더 `start` 후 다시 확인합니다.
3. 그 다음 `./scripts/browser-mcp/browser-ensure-ready-cs.sh`를 실행해 브라우저 준비를 확인합니다.
4. 열린 탭 목록을 먼저 확인하고 `sell.smartstore.naver.com` 또는 스마트스토어 관련 URL 탭이 이미 열려 있으면 그 탭 하나를 선택해 끝까지 재사용합니다.
5. 기존 스마트스토어 탭이 있으면 절대 새 탭을 열지 말고, 그 탭 안에서 대시보드/문의/Q&A 페이지로 이동합니다.
6. 스마트스토어 관련 탭이 전혀 없을 때만 새 탭을 1개 열 수 있습니다.
7. 스마트스토어 대시보드 또는 문의 화면으로 진입합니다.
8. 로그인되지 않은 상태면 로그인 버튼을 클릭하지 말고, 로그인 필요 상태로만 짧게 보고하고 중단합니다.
9. 멀티스토어 선택이 가능하면 `스토어 이동`을 눌러 `또몽이네 스토어`를 선택합니다.
10. 브라우저 제어가 attach 승인 대기에서 멈춘 흔적이 보이면, 다른 우회 없이 `./scripts/browser-mcp/chrome-dev-approve-session.sh start` → `./scripts/browser-mcp/browser-ensure-ready-cs.sh` 순서로 1회 재실행한 뒤에만 계속 진행합니다.

[2] 대시보드 건수 확인
1. 스토어 대시보드의 미처리 문의 영역에서 아래 3개 항목의 건수를 확인합니다.
   - 톡톡 문의
   - 주문 고객 문의
   - 상품 Q&A
2. `./tmp/cs-monitor-state.json` 파일을 읽어 직전 보고값과 비교합니다. 파일이 없거나 대상 스토어 기록이 없으면 직전값은 0으로 봅니다.
3. 신규 증가 건수는 `(현재 건수 - 직전 건수)`로 계산하되 음수면 0으로 처리합니다.
4. 신규 CS 합계는 `신규 톡톡 + 신규 주문문의 + 신규 상품 Q&A` 입니다.

[3] 상품 Q&A 초안 생성
1. 상품 Q&A 현재 건수가 1건 이상이면 상품 Q&A 페이지로 들어가 미답변 목록을 확인합니다.
2. 미답변 항목마다 상품명, 질문 원문을 추출하고 고객에게 보낼 수 있는 답변 초안을 작성합니다.
3. 초안은 또몽이네 스토어 CS 규칙을 따르고 반드시 code block으로 작성합니다.
4. 자동 등록은 하지 말고 초안 생성까지만 합니다.

[4] 보고 규칙
1. 세 항목이 모두 0건이면 현재 Discord 채널에 아래 형식으로만 보고하고 종료합니다.
```text
또몽이네 스토어 CS 점검 완료: 처리할 문의 없음
```
2. 처리할 문의가 있으면 현재 Discord 채널에 아래 형식으로 보고합니다.

[또몽이네 스토어 CS 처리 필요]
- 현재 미처리: 톡톡 N건 / 주문 고객 문의 N건 / 상품 Q&A N건
- 신규 증가: 톡톡 +N건 / 주문 고객 문의 +N건 / 상품 Q&A +N건
- 신규 CS 합계: N건
- 메모: 대시보드 기준 미처리 문의가 있어 세부 대응이 필요합니다.

3. 상품 Q&A 초안이 있으면 이어서 아래 형식으로 추가 보고합니다.

[또몽이네 스토어 상품 Q&A 초안]
1번
- 상품명: "상품명"
- 질문: "질문 원문"
- 초안:
```text
안녕하세요, 또몽이네 스토어입니다.

문의주신 내용에 대한 답변 문장을 여기 완성형으로 작성합니다.

감사합니다.
```

2번
- 상품명: "상품명"
- 질문: "질문 원문"
- 초안:
```text
안녕하세요, 또몽이네 스토어입니다.

문의주신 내용에 대한 답변 문장을 여기 완성형으로 작성합니다.

감사합니다.
```

[5] 상태 저장
1. 보고가 끝나면 `./tmp/cs-monitor-state.json`에 또몽이네 스토어의 현재 건수와 보고 시각을 저장합니다.
2. 기존 다른 스토어 기록은 유지하고, 대상 스토어 항목만 갱신합니다.

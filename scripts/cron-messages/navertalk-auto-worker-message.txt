톡톡 웹훅 자동 루프 작업입니다.

반드시 아래 순서대로 진행하세요.

[1] 로컬 설정 읽기
1. `./runtime-data/navertalk-local-config.json` 파일을 읽습니다.
2. 여기서 아래 값을 사용합니다.
   - `viewerToken`
   - `approvalChannelId`
   - `controlChannelId`
3. 파일이 없거나 값이 비어 있으면 현재 세션에 짧게 실패 사유를 보고하고 종료합니다.

[2] approval / outbox 생성
1. 아래 명령을 실행합니다.
```bash
NAVERTALK_VIEWER_TOKEN="<viewerToken>" NAVERTALK_APPROVAL_CHANNEL_ID="<approvalChannelId>" node ./scripts/navertalk-auto-loop.mjs
```
2. 실행 결과를 읽고 아래를 확인합니다.
   - `createdCount`
   - `syncResult.createdCount`
3. 새 approval / outbox 생성이 0이고, 아래 [3] 발송 대상도 없으면 `NO_REPLY`로 종료합니다.

[3] pending_send outbox 발송
1. `./runtime-data/local-cs-discord-outbox/` 에서 `.json` 파일을 읽습니다.
2. `state = pending_send` 인 항목만 발송 대상으로 잡습니다.
3. `meta.reason = superseded_by_newer_short_code` 이 있거나 `state = superseded` 인 파일은 무시합니다.
4. 발송 대상이 여러 건이면 `createdAt` 오래된 순으로 처리합니다.
5. 각 outbox 의 `sendRequest` 를 그대로 사용해 Discord 승인 채널로 보냅니다.
   - `channel = discord`
   - `target = channel:<approvalChannelId>`
   - 버튼 UI 포함
6. 발송 성공 시 아래 명령으로 상태를 `sent` 로 갱신합니다.
```bash
node ./scripts/navertalk-outbox-mark-sent.mjs <approvalId> sent <messageId> <approvalChannelId>
```
7. 발송 실패 시 아래 명령으로 상태를 `failed` 로 남깁니다.
```bash
node ./scripts/navertalk-outbox-mark-sent.mjs <approvalId> failed
```

[4] 응답 규칙
1. 승인 카드를 실제로 보냈다면 최종 응답은 `NO_REPLY` 입니다.
2. 처리할 것이 아무것도 없으면 `NO_REPLY` 입니다.
3. 설정 파일 누락이나 반복 실패 같은 실제 장애만 현재 세션에 짧게 보고합니다.
